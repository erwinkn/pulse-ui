---
title: QueryConfig
---

Configuration options for a query.

Stores immutable configuration that controls query behavior including
retry logic, caching, and lifecycle callbacks.

## Attributes

<PyAttribute name={"retries"} type={"int"} value={null}>

Number of retry attempts on failure (default 3).

</PyAttribute>

<PyAttribute name={"retry_delay"} type={"float"} value={null}>

Delay in seconds between retry attempts (default 2.0).

</PyAttribute>

<PyAttribute name={"initial_data"} type={"T | Callable[[], T] | None"} value={null}>

Initial data value or factory function.

</PyAttribute>

<PyAttribute name={"initial_data_updated_at"} type={"float | dt.datetime | None"} value={null}>

Timestamp for initial data staleness calculation.

</PyAttribute>

<PyAttribute name={"gc_time"} type={"float"} value={null}>

Seconds to keep unused query in cache before garbage collection.

</PyAttribute>

<PyAttribute name={"on_dispose"} type={"Callable[[Any], None] | None"} value={null}>

Callback invoked when query is disposed.

</PyAttribute>
