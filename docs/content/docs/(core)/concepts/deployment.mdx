Pulse always uses one browser-visible origin. You can still scale the Python backend and React Router server independently behind a proxy or ingress.

## The request boundary

The browser only sees your public origin:

```text
Browser -> https://app.example.com -> proxy / ingress
                                      |-> Pulse backend
                                      |-> React Router server
```

Route these paths to the Pulse backend:

- `/_pulse/*`, including `/_pulse/socket.io`
- your FastAPI routes, such as `/api/*` and `/auth/*`

Route page and asset requests to React Router. Keep the `/_pulse` rule ahead of any frontend catch-all. Pulse's framework prefix is fixed and reserved.

The browser uses relative URLs and same-origin credentials. Pulse does not configure CORS for its framework transport; the Socket.IO server only accepts connections whose `Origin` matches the request host. A proxy or ingress in front of Pulse must therefore forward the original `Host` header (or set `X-Forwarded-Host`) and set `X-Forwarded-Proto` when it terminates TLS — standard behavior for most proxies. If yours cannot, override the check with `App(socketio_options={"cors_allowed_origins": ...})`.

## Run both servers together

```bash
pulse run app.py --prod --address 0.0.0.0
```

This starts both processes. Pulse receives browser traffic and proxies unmatched HTTP and WebSocket requests to the internal React Router server (in development, that includes Vite's HMR socket). The upstream sees its own internal host; the browser-visible host and scheme are forwarded as `X-Forwarded-Host` and `X-Forwarded-Proto`, so remote access through tunnels like Tailscale or ngrok works without extra Vite configuration.

The CLI sets the internal addresses automatically. `WebProxyConfig` controls the optional web proxy:

```python
import pulse as ps

app = ps.App(
    routes=[...],
    proxy=ps.WebProxyConfig(max_concurrency=200),
)
```

Pulse constructs the internal `WebProxy` only when `PULSE_WEB_UPSTREAM` is set. Without that variable, the server is backend-only.

## Scale the processes separately

Run the Pulse backend without a web proxy:

```bash
pulse run app.py --prod --backend-only --address 0.0.0.0
```

Run React Router separately and give its server-side loaders an internal backend URL:

```bash
pulse run app.py --prod --web-only \
  --ssr-backend-url http://pulse-backend:8000
```

`PULSE_SSR_BACKEND_URL` is private server-to-server configuration. It is read by React Router at runtime, so generated files and frontend builds remain origin-neutral.

If you run the ASGI app outside the Pulse CLI, configure process composition directly:

```bash
PULSE_ENV=prod \
PULSE_WEB_UPSTREAM=http://react-router:3000 \
uvicorn app:app.asgi_factory --factory --host 0.0.0.0 --port 8000
```

Omit `PULSE_WEB_UPSTREAM` for a backend-only process.

## Public origin

Most Pulse apps do not need a configured public origin. Set one only when server-side code must create an absolute callback URL, such as an OAuth redirect:

```python
import pulse as ps

app = ps.App(
    routes=[...],
    public_origin="https://app.example.com",
)
```

`PULSE_PUBLIC_ORIGIN` provides the same runtime setting. An explicit `App(public_origin=...)` takes precedence. The value must be an HTTP or HTTPS origin with no path, query, credentials, or fragment; production and CI require HTTPS.

Do not use the public origin for browser fetches. Use relative paths such as `/api/users`.

## Cookies

Pulse session cookies are host-only: Pulse never sets a `Domain` attribute. This matches the single-origin boundary and prevents sibling hosts from receiving session cookies.

The default cookie is `Secure` in production and CI. Pulse rejects an explicitly insecure cookie in those environments. Development resolves `secure` from `public_origin` when one is configured.

```python
app = ps.App(
    routes=[...],
    cookie=ps.Cookie(
        "pulse.sid",
        secure=True,
        samesite="lax",
        max_age_seconds=30 * 24 * 3600,
    ),
)
```

## Environment variables

| Variable | Purpose | Example |
| --- | --- | --- |
| `PULSE_ENV` | Runtime environment | `prod` |
| `PULSE_PUBLIC_ORIGIN` | Optional canonical browser origin | `https://app.example.com` |
| `PULSE_WEB_UPSTREAM` | Optional internal React Router URL for Pulse's web proxy | `http://react-router:3000` |
| `PULSE_SSR_BACKEND_URL` | Internal Pulse URL used by React Router SSR | `http://pulse-backend:8000` |
| `PULSE_HOST` | Pulse bind host, set by the CLI | `0.0.0.0` |
| `PULSE_PORT` | Pulse bind port, set by the CLI | `8000` |

## Code generation

Code generation does not take public or internal addresses:

```bash
pulse generate app.py --prod
```

```python
app.run_codegen()
```

Generated browser requests target the current origin. The React Router server reads `PULSE_SSR_BACKEND_URL` when it handles a request.
