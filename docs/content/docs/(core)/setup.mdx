Welcome to Pulse! This guide will walk you through creating your first Pulse application from scratch.

<Callout type="warn" title="Early access">
Pulse is under active development and this setup process will be simplified in the future. For now, you'll need to set up two parts: a Python project (where your app logic lives) and a small Vite web project (which serves the frontend).
</Callout>

## What you'll need

Before getting started, make sure you have these tools installed:

- **[uv](https://docs.astral.sh/uv/)** — A fast Python package manager
- **[Bun](https://bun.sh/)** — A JavaScript runtime and package manager

## Step 1: Create your Python project

First, let's set up a new directory for your Pulse app and initialize it as a Python project:

```bash
mkdir my-pulse-app
cd my-pulse-app
uv init
uv add pulse-framework
uv sync
```

This creates a new Python project managed by uv and installs the Pulse framework.

Now create a file called `app.py` with your first Pulse component:

```python
from pathlib import Path
import pulse as ps

@ps.component
def home():
    return ps.div("Hello Pulse")

app = ps.App(
    routes=[ps.Route("/", home)],
    codegen=ps.CodegenConfig(web_dir=Path(__file__).parent / "web"),
)
```

Let's break down what's happening here:

- **`@ps.component`** — This decorator turns a Python function into a Pulse component. Components are the building blocks of your UI.
- **`ps.div("Hello Pulse")`** — This creates a `<div>` element with the text "Hello Pulse" inside it. Pulse provides functions like `div`, `span`, `button`, etc. that map to HTML elements.
- **`ps.App`** — This is your application object. It holds your routes and configuration.
- **`ps.Route("/", home)`** — This tells Pulse to render the `home` component when someone visits the root URL (`/`).
- **`CodegenConfig(web_dir=...)`** — This tells Pulse where your React frontend lives (we'll create that next).

## Step 2: Create the web project

Pulse ships its own React framework—a plain Vite project plus a small Bun server that renders HTML. There's no scaffolding command yet, so the easiest way to get started is to copy the [`examples/web/`](https://github.com/erwinkn/pulse-ui/tree/main/examples/web) folder from the Pulse repository into your project as `web/`, then install its dependencies:

```bash
cd web
bun install
cd ..
```

Here's what lives in that folder—these files are yours to edit:

```
web/
  index.html              # HTML shell for the Vite dev server
  vite.config.ts          # Plain Vite config with @vitejs/plugin-react
  src/
    entry-client.tsx      # Hydrates the app in the browser
    entry-server.tsx      # Renders HTML on the server
  server/
    ssr.ts                # Small Bun server that serves the rendered HTML
  app/
    pulse/                # Generated by Pulse — don't edit, it's rebuilt on every run
```

The `app/pulse/` directory is where Pulse writes the generated route files that connect your Python components to React. Everything else is a regular Vite + React project, so you can add global providers, CSS frameworks, or extra dependencies the way you would in any React app.

Good news: you don't need to manage the `pulse-ui-client` version yourself. When you run `pulse run` in development, Pulse checks the web project's dependencies and installs the version matching your Python package automatically.

## Step 3: Run your app

Now you're ready to start your Pulse app:

```bash
uv run pulse run app.py
```

The `pulse run` command does three things:
1. **Generates routes** — It reads your Python route definitions and creates corresponding files in `web/app/pulse/`
2. **Starts the web servers** — A Vite dev server for assets and hot reloading, plus a Bun server that renders HTML
3. **Starts the Python server** — Your Pulse app, served with uvicorn

Open the URL shown in your terminal (usually `http://localhost:8000`) and you should see "Hello Pulse" displayed in your browser!

## Customizing the frontend location

By default, Pulse expects your React app to be in a `web/` folder next to your Python file. If your frontend lives somewhere else, just update the `web_dir` setting:

```python
codegen=ps.CodegenConfig(web_dir=Path("/path/to/your/react-app"))
```
