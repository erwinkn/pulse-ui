# Serialization

Pulse uses one versioned format for browser-server messages. It preserves
`Date`, `Set`, `Map`, shared references, and cycles while remaining valid JSON.

## serialize

```ts
import { serialize } from "pulse-ui-client";

function serialize(data: unknown): Serialized;
type Serialized = [5, WireValue];
```

```ts
const value = {
  createdAt: new Date("2026-07-16T12:30:00.000Z"),
  tags: new Set(["active", "admin"]),
  metadata: new Map([["version", 5]]),
};

serialize(value);
// [5, {
//   createdAt: ["$", "t", "2026-07-16T12:30:00.000Z"],
//   tags: ["$", "s", ["active", "admin"]],
//   metadata: ["$", "m", [["version", 5]]]
// }]
```

### Supported values

| JavaScript value | Wire behavior |
|---|---|
| `null`, boolean, string | Preserved |
| finite number | Preserved; `-0` becomes `0`; magnitudes beyond 2^53 use the big-float marker |
| `NaN` | Becomes `null` |
| `Date` | UTC timestamp marker |
| `Array` | Preserved, including shared identity and cycles |
| plain or null-prototype object | Encoded as a record |
| `Map<string, value>` | Insertion order preserved |
| `Set` | Canonically ordered |

Strings and record keys must contain valid Unicode scalar values. Infinity,
invalid Dates, functions, symbols, BigInt, class instances, DOM objects,
File/Blob, buffers, typed arrays, RegExp, and URL are rejected. Doubles with
magnitude beyond 2^53 travel as `["$", "f", value]` so Python decodes them as
`float` rather than `int`; plain wire numbers outside the safe integer range
are rejected.

Array holes and `undefined` array entries become `null`. Extra, symbol, and
non-enumerable array properties are ignored. Plain-record getters can execute
once. Proxies are unsupported.

### undefined and NaN

Normalization happens during the one-pass encoder:

```ts
serialize({ omitted: undefined, kept: null });
// [5, { kept: null }]

serialize([undefined, NaN]);
// [5, [null, null]]

serialize(new Map([["value", undefined]]));
// [5, ["$", "m", [["value", null]]]]
```

Root `undefined`, array holes/items, and Map values become `null`. An undefined
record property is omitted. Set entries normalize before collision checks, so a
Set containing both `undefined` and `null`, or both `NaN` and `null`, fails.

## deserialize

```ts
import { deserialize } from "pulse-ui-client";

function deserialize(payload: unknown): unknown;
```

`null` always remains `null`; there is no null-to-undefined option. Timestamp
markers become native `Date` values. Map and Set markers become their native
JavaScript types. Shared references and cycles are restored.

The decoder accepts JSON/Socket.IO-materialized values and still validates every
scalar, marker, reference, Map, Set, and record key. `__proto__` is assigned as
an own data property without changing the output object's prototype.

## Wire markers

| Value | Marker |
|---|---|
| Escaped array beginning with `"$"` | `["$", "a", items]` |
| UTC timestamp | `["$", "t", isoTimestamp]` |
| Double beyond 2^53 | `["$", "f", value]` |
| Map | `["$", "m", entries]` |
| Set | `["$", "s", items]` |
| Back reference | `["$", id]` |

References point only to an earlier identity and use non-negative safe integer
IDs. An ordinary array beginning with `"$"` uses the escaped-array marker so it
cannot be mistaken for a marker.

Python `date` and aware `datetime` values both arrive as `Date`. A `Date` sent to
Python becomes an aware UTC `datetime`; calendar-only type information is
intentionally not preserved.
