Pulse uses a versioned wire serializer for component props, callback arguments,
forms, and channels. The current envelope is `[5, value]`.

## Default serializer

```python
import pulse as ps

payload = ps.serialize({"items": [1, 2, 3]})
value = ps.deserialize(payload)
```

`ps.serialize()` and `ps.deserialize()` use one immutable default
`ps.Serializer`. Use an app-owned serializer when you need adapters.

## Serializer

```python
class Serializer:
    def __init__(
        self,
        adapters: Iterable[SerializerAdapter[Any]] = (),
    ) -> None: ...

    def serialize(self, value: object) -> Serialized: ...
    def deserialize(self, payload: object) -> object: ...
```

`ps.App(serializer=...)` uses that serializer for the app's render sessions,
callbacks, forms, and messages. Adapter registries cannot be changed after the
serializer is created.

## Supported values

| Python value | Browser value | Notes |
|---|---|---|
| `None` | `null` | Round-trips exactly |
| `bool`, `str` | Same primitive | Strings must contain valid Unicode scalar values |
| finite `float` | `number` | `NaN` becomes `null`; `-0.0` becomes `0.0`; infinity is rejected; magnitudes beyond 2^53 use the big-float marker |
| safe `int` | `number` | Range is `-(2**53 - 1)` through `2**53 - 1` |
| `list`, `tuple` | `Array` | Tuples decode as lists |
| `dict[str, object]` | plain object | Record keys follow JavaScript object-key order |
| `WireMap` | `Map` | String keys, insertion order preserved |
| `set` | `Set` | Canonically ordered; cross-runtime equality collisions fail |
| `date`, aware `datetime` | `Date` | Normalized to UTC with exact millisecond precision |
| dataclass instance | plain object | Field order is preserved |

Arbitrary objects, callables, classes, modules, naive datetimes, sub-millisecond
datetimes, infinity, unsafe integers, and invalid Unicode are rejected.

### Dates and datetimes

```python
from datetime import date, datetime, timezone

ps.serialize(date(2026, 7, 16))
# [5, ["$", "t", "2026-07-16T00:00:00.000Z"]]

ps.serialize(datetime(2026, 7, 16, 12, 30, tzinfo=timezone.utc))
# [5, ["$", "t", "2026-07-16T12:30:00.000Z"]]
```

Both values become a native JavaScript `Date`. A `Date` sent back to Python
always becomes an aware UTC `datetime`. The original `date`/`datetime`
distinction is intentionally lost. Use an ISO string when calendar-only type
preservation matters.

Python datetimes must be timezone-aware and have exact millisecond precision.
Offsets are normalized to UTC. Years outside `0001` through `9999` fail.

### Maps

```python
import pulse as ps

value = ps.WireMap([("first", 1), ("second", 2)])
```

Normal dictionaries are records. `WireMap` explicitly requests the wire-level
Map marker and decodes back to `WireMap` in Python.

## Adapters

```python
@dataclass(frozen=True, slots=True)
class SerializerAdapter[T]:
    type: type[T]
    serialize: Callable[[T], object]

class PulseSerializable(ABC):
    @abstractmethod
    def to_pulse(self) -> object: ...
```

An adapter is one-way: it projects an application value into supported Pulse
values. Deserialization returns the projection, not the original class.

Resolution order is:

1. Exact core wire type
2. Nearest registered adapter in the value's MRO
3. `PulseSerializable`
4. Dataclass
5. Supported container subclass

Configured adapters override `PulseSerializable` and dataclass behavior.
Duplicate targets and adapters for `object` or core wire types are rejected.

Structural projections share the source object's wire identity, so repetition
and self-cycles survive. Scalar projections use value semantics. Returning the
source object directly is an error.

See [Custom serialization](/docs/cookbook/advanced-patterns/custom-serialization)
for a complete example.

## Wire markers

| Value | Marker |
|---|---|
| Escaped array beginning with `"$"` | `["$", "a", items]` |
| UTC timestamp | `["$", "t", iso_timestamp]` |
| Map | `["$", "m", entries]` |
| Set | `["$", "s", items]` |
| Back reference | `["$", id]` |

Reference IDs are backward-only non-negative safe integers. The decoder rejects
unknown and malformed markers, dangling references, duplicate Map keys,
non-canonical Sets, and equality collisions.
