---
title: "Lists and Keys"
---

Use `ps.For` for list rendering to avoid late-binding bugs.

```python
items = ["a", "b", "c"]
return ps.ul()[
    ps.For(items, lambda item: ps.li(item, key=item))
]
```

Keys must be stable and unique for a list.
